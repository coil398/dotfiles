# 書式ルール

このファイルは `~/.claude/CLAUDE.md` から参照される AI 応答の書式ガイドライン。AI はこのファイルに書かれたルールに従って応答を生成する。

---

## 大原則

- **情報量は減らさない**。短くしようとして必要な選択肢・背景・参照を削らない
- **視覚構造で読みやすくする**。ターミナルがレンダリングする要素（見出しの色、シンタックスハイライト、引用バー、表罫線等）を意図的に使い分ける
- **色の多様性を確保する**。単調な段落は避け、最低 2 種類以上の色つき要素（見出し + コードブロック、表 + 引用バー、diff + 絵文字バッジ等）を混ぜる

---

## 1. 見出しレベルの使い分け

意味論的に階層を揃える。ターミナルは `#` / `##` / `###` / `####` で異なる色・太さでレンダリングするので、深さ = 視覚的重みとして機能する。

| レベル | 用途 | 例 |
|--------|------|-----|
| `#` | 応答全体の主見出し（1 応答に 1 個まで） | `# セッション結果` |
| `##` | 章（複数のサブトピックをまとめる単位） | `## 実装した変更` |
| `###` | 節（章内の個別トピック） | `### Edit 1 — pir2async 修正` |
| `####` | 小節（節内の補足ポイント） | `#### 修正理由` |

- 見出しは**必ず章立てに使う**。本文の強調目的では使わない
- 1 応答内に `##` が 1 つだけで終わる場合は、`##` を使わず本文だけで良い
- `#` を章と同じレベルで乱用しない。最上位として 1 個まで

---

## 2. コードブロックは必ず言語タグを付ける

```` ```言語 ```` でシンタックスハイライトが効く。無タグの ```` ``` ```` だけでは色がつかない。

| 言語タグ | 使う場面 |
|----------|----------|
| ` ```diff ` | **差分・変更前後の表示は必須**。行頭 `-` が赤、`+` が緑でレンダリングされる |
| ` ```json ` | `settings.json`・API ペイロード・設定ブロック |
| ` ```bash ` / ` ```sh ` | シェルコマンド・実行例 |
| ` ```ts ` / ` ```tsx ` / ` ```go ` / ` ```py ` 等 | 該当言語のコード片 |
| ` ```yaml ` / ` ```toml ` | 設定ファイル |
| ` ```md ` | Markdown サンプル |

### 差分を見せるときのルール（厳守）

ファイルを編集したり設定を追加/削除したりしたら、**必ず ` ```diff ` ブロックで before/after を提示する**。文章だけで「`foo` を `bar` に変えました」と済ませない。

```diff
- "Edit(*/.claude/projects/*/memory/**)",
+ "Edit(**/.claude/projects/*/memory/**)",
```

---

## 3. インラインコード

`` `foo` `` を使うのは以下の用途のみ:

- 識別子（変数名・関数名・クラス名）
- ファイルパス・ディレクトリ名
- コマンド断片・CLI フラグ
- 設定キー・環境変数
- 技術用語のうち「リテラルに解釈してほしい」語

日本語の地の文に混ぜるだけで視覚的に浮いて読みやすくなる。自然文の中の強調目的では使わない。

---

## 4. 引用ブロック `>` は注記バッジとして使う

行頭に絵文字を置いて注記の種別を明示する。ターミナルは引用バーに色を付けるので、バッジとの組み合わせで種別が一目でわかる。

| バッジ | 用途 |
|--------|------|
| `> ℹ️` | 参考情報・補足 |
| `> ⚠️` | 注意・落とし穴・制約 |
| `> ✅` | 成功・確認済み・推奨 |
| `> ❌` | 非推奨・禁止・失敗 |
| `> 🔧` | 手動設定・後処理が必要 |
| `> 📝` | メモ・議事録的な記録 |
| `> 💡` | ヒント・代替案 |

例:

> ⚠️ Claude Code の permission glob は `*` が `/` を跨がない。絶対パスをマッチさせたいなら `**/` を使うこと。

> ✅ `diff` ブロックは緑/赤でレンダリングされるので、差分提示ではこれを第一選択にする。

---

## 5. 絵文字バッジの使い分け

引用ブロック以外でも、**セクション見出しや箇条書きの先頭に絵文字バッジを置いて種別を示してよい**。ただし乱用禁止 — 1 セクション 1 バッジまで、装飾目的では付けない。

### 推奨セット

| バッジ | 意味 |
|--------|------|
| 📋 | まとめ・一覧 |
| 🔍 | 調査・探索結果 |
| 🎯 | 今回の目的・ゴール |
| ✨ | 新機能・新規追加 |
| 🐛 | バグ・不具合 |
| 🔧 | 設定・構成変更 |
| 🚀 | デプロイ・リリース |
| 📊 | 集計・統計 |
| ❓ | 質問・未解決事項 |
| ⏭️ | 次のアクション |

---

## 6. 表で一覧化する

3 項目以上の並列情報（候補比較・Before/After・Counts・ファイル一覧等）は**必ず表にする**。箇条書きを平らに並べない。

```md
| # | 項目 | 値 | 備考 |
|---|------|-----|------|
| 1 | foo | 10 | 通常 |
| 2 | bar | 20 | 要注意 |
```

---

## 7. 強調記法は「本当に外せない」点だけ

`**太字**` / `*斜体*` は、応答全体で 1〜3 箇所に絞る。

- ✅ 良い例: **必ず**・**禁止**・**この順番で**
- ❌ 悪い例: 1 段落に 3 回以上太字が出る / 全ての箇条書き項目の先頭が太字

全部に付けると強調差が消える。太字は「読み飛ばされたら重大な誤解が起きる」一点に絞る。

---

## 8. NG パターン

| NG | 改善 |
|----|------|
| 地のテキストだけの長い段落が連続 | 章立て + 表 + コードブロックを混ぜる |
| ` ``` ` に言語タグなし | 必ず ` ```diff ` / ` ```json ` 等を付ける |
| 差分を文章で説明（「`foo` を `bar` に変えた」だけ） | `diff` ブロックで before/after を提示 |
| 太字だらけ | 1 応答 1〜3 箇所まで |
| 絵文字なしで平坦な箇条書きが 10 項目以上続く | 表に変換、または小見出しで分割 |
| 引用ブロックに種別バッジなし | `> ℹ️` `> ⚠️` 等でラベル付け |

---

## 9. 応答構造テンプレート

中〜大規模な応答の基本骨格:

```md
# 🎯 ゴール

1 行で今回の応答目的。

---

## 📋 変更内容

### Edit 1 — <対象>

\```diff
- 旧
+ 新
\```

> ℹ️ 変更理由の補足

### Edit 2 — <対象>

...

---

## 🔍 確認事項

| # | 確認点 | 状態 |
|---|--------|------|
| 1 | ... | ✅ |

---

## ⏭️ 次のアクション

- A 案
- B 案
```

小規模な応答（1〜2 行で済むもの）はテンプレートに縛られる必要はない。**タスクに見合う粒度で**。質問に一言で答えるだけのときは装飾を増やさない。

---

## 10. このルールが適用される範囲

- Claude Code のチャット応答全般
- `docs/brainstorm/`, `docs/walkthrough/`, `docs/plans/` 等の成果物ファイル
- コミットメッセージや PR 説明（ただし GitHub 側の Markdown レンダリング仕様に準拠）

**適用されない範囲**:
- ソースコードのコメント（無駄に絵文字を入れない）
- テスト出力・ログ出力
