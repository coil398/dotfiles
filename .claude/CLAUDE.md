# Claude Code 設定

## 出力言語
- すべての出力・回答は**日本語**で行う

## グローバルファイルの汎用性ルール

グローバルスコープのファイル（`~/.claude/CLAUDE.md`、`~/.claude/agents/`、`~/.claude/skills/`）は **全プロジェクト共通** で使われる。特定プロジェクトに依存する内容を混入させてはならない。

- **禁止**: プロジェクト固有の名前（クラス名、テーブル名、API エンドポイント名、`FooUsecase` のような具体名）
- **許可**: 汎用的な概念（「usecase」「repository」「controller」等のパターン名）や例示としての仮名（`XxxService` 等）
- プロジェクト固有の情報はそのプロジェクトの `.claude/CLAUDE.md` またはプロジェクトメモリ（`~/.claude/projects/` 配下）に書くこと
- retrospector がエージェント定義を更新する際もこのルールを厳守すること

## tech-validator エージェントの使用ルール

### 使うタイミング
以下の状況では **必ず** `tech-validator` エージェントを起動してからライブラリを決定すること：

- 新しいライブラリ・フレームワークをプロジェクトに追加するとき
- 既存依存関係の更新・置き換えを検討するとき
- 同種のライブラリが複数存在し、どれを選ぶか判断が必要なとき
- 「○○を実現するのに適したライブラリは？」と聞かれたとき

### 禁止事項
- バージョンを記憶や推測で記載しない（必ず `npm show` で確認）
- tech-validator を経ずに独断でライブラリを選定しない
- 選定レポートなしに `npm install` を提案しない
