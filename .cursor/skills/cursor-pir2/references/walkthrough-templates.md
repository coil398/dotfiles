# ウォークスルー生成テンプレート

PIR² 系スキル（/pir2）のステップ 9「ウォークスルー生成」で使うテンプレートと原則。フル版（内部記録 / 実装記録ドキュメントの「実装ログ」に埋める）とサマリー版（最終サマリーに転記）の 2 形式を作成する。

## フル版（内部記録）

各ファイルについて以下を記述する:

```
#### [ファイルパス]
- 何を変更したか: [追加・変更・削除した内容]
- 主要なコード:
  [変更の核となるコード片を引用]
- なぜこの実装にしたか: [既存パターンとの整合性、トレードオフ、代替案を採用しなかった理由]
```

実装記録ドキュメント（ステップ 5 で作成したファイル）の「実装ログ」セクションに埋める。

## サマリー版（統合レポートに載せる）

```
### ウォークスルー（サマリー版）

#### 変更の全体像
[30秒で把握できるよう3〜4文で。「何ができるようになったか」「何が直ったか」中心]

#### ファイルごとの変更
- `path/to/file1` — [1〜2行。必要なら核心のコード片を最大5行]
- `path/to/file2` — [同上]

#### 設計判断（非自明なもののみ）
[代替案がある中でなぜこの方法を選んだか。自明なら「特になし」]

#### レビュー総括
[何を確認し、なぜ PASS/FAIL としたか。簡潔に]
```

## サマリー版の原則

- 箇条書きは 1 階層まで。ネストしない
- コード片引用は変更の核心のみ、最大 5 行程度
- 「なぜ」は非自明な判断のみ
- **推測でコードを書かない。実際に Read したコードのみ引用する**
